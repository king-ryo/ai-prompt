# 講義用Webアプリ作成プロンプト（完全版）

## 役割

あなたは熟練したフロントエンドエンジニア兼UIデザイナーです。
添付のMarkdownファイル（講義資料）をもとに、ブラウザで動作する高品質な「アクティブラーニング用Webアプリケーション」を作成してください。

## 1. ファイル構成と技術スタック

* **技術:** HTML5, CSS3, JavaScript (Vanilla JS) のみ。フレームワーク（React/Vue等）は使用しない。
* **ファイル分割:** 以下の3ファイルにコードを完全に分離して出力すること。
1. `index.html` (構造)
2. `style.css` (デザイン)
3. `script.js` (ロジック)


* **アーキテクチャ:** 画面遷移（リロード）を伴わず、JavaScriptでDOMクラス（`.hidden`等）を操作してビューを切り替えるシングルページアプリケーション（SPA）構成とする。

## 2. デザイン・レイアウト要件

* **ブランドカラー:** `#2ECCB0`（エメラルドグリーン）を基調とする。
* **デザインスタイル:**
* 清潔感のあるフラットデザイン。十分な余白（マージン/パディング）を取り、視認性を重視する。
* テキストはダークグレー、背景はオフホワイトまたは白。


* **画面構成:**
* **講義ビュー（3カラム）:**
* 左：章ナビゲーション（現在地ハイライト）。
* 中央：メインコンテンツ（スライド・ワーク・クイズ）。
* 右：目次（ページ内リンク）。


* **学習完了ビュー（1カラム・中央配置）:**
* 学習完了時に表示する専用画面。
* 達成感を演出するデザイン（トロフィーアイコン、祝福メッセージ、アニメーションなど）。





## 3. 機能・挙動要件

### A. プログレッシブ・ディスクロージャー（段階的表示）

* 各章のコンテンツは一度に表示せず、情報の塊（ステップ）ごとに「次へ進む」ボタンでフェードイン表示させる。
* 章の最後に到達したら「次の章へ」ボタンで遷移させる。

### B. インタラクティブ機能（モーダル・クイズ）

* **用語解説モーダル:**
* Markdown内の重要語句（「IaC」「イメージ」「Nginx」等）はリンク化し、クリック時にモーダルで解説を表示する。
* **解説内容の質:** 初心者向けの比喩だけでなく、**「バージョン管理」「再現性（冪等性）」**などのエンジニアリング用語を用いた、実務に即した専門的な解説文を実装すること。


* **確認テスト（クイズ）:**
* Markdown内の「確認問題」や「振り返り」箇所は、**「4択クイズ形式」**で実装する。
* 選択肢をクリックした瞬間に「正解/不正解」の判定と解説を表示する（即時フィードバック）。



### C. 学習完了体験（ゲーミフィケーション）

* 最終章の学習が終了したら、「学習完了」ボタンを表示する。
* ボタン押下時はアラートではなく、**「学習完了ビュー」へ画面全体を切り替える。**
* 完了画面には「トップに戻る（リスタート）」ボタンを配置し、学習を最初からやり直せるようにする。

### D. スライド表示（PDF）

* 各章の終了時などに「まとめスライドを表示」ボタンを配置し、押下時にモーダルでPDF（例: `slide/chapter1.pdf`）を表示可能にする。

## 4. コンテンツの反映

* 添付の `01-基礎編.md` の内容を漏らさず実装すること。
* Markdownの階層構造をHTMLに適切に変換し、数式がある場合はMathJax等でレンダリングすること。

## 出力のお願い

解説は最小限にし、コピー＆ペーストですぐに動作確認ができる完全な `index.html`, `style.css`, `script.js` のソースコードを提示してください。

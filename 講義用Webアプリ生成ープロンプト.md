# 講義用Webアプリ作成プロンプト

## 役割

あなたは熟練したフロントエンドエンジニア兼UIデザイナーです。
添付のMarkdownファイル（講義資料）をもとに、ブラウザで動作する「アクティブラーニング用Webアプリケーション」を作成してください。

## 1. ファイル構成と技術スタック

* **技術:** HTML5, CSS3, JavaScript (Vanilla JS) のみ。フレームワーク（React/Vue等）は使用しない。
* **ファイル分割:** 以下の3ファイルにコードを完全に分離して出力すること。
1. `index.html` (構造)
2. `style.css` (デザイン)
3. `script.js` (ロジック)


* **アーキテクチャ:** 画面遷移（リロード）を伴わず、JavaScriptでDOMを操作してコンテンツを切り替えるシングルページアプリケーション（SPA）構成とする。

## 2. デザイン・レイアウト要件

* **ブランドカラー:** `#2ECCB0`（エメラルドグリーン）をメインカラーとして使用する。
* **デザインスタイル:**
* 清潔感のあるフラットデザイン。
* 圧迫感を避けるため、十分な余白（マージン/パディング）を取り、視認性とバランスを意識したレイアウトにする。
* テキストは読みやすいダークグレー、背景はオフホワイトまたは白を基調とする。


* **画面レイアウト（3カラム構成）:**
* **左サイドバー:** 章ごとのナビゲーション（現在地をハイライト）。
* **中央:** メインコンテンツエリア（講義スライド・ワーク表示部）。
* **右サイドバー:** 現在開いている章の「目次（小見出し）」へのショートカットリンク。



## 3. 機能・挙動要件

* **プログレッシブ・ディスクロージャー（段階的表示）:**
* 各章のコンテンツは一度にすべて表示せず、情報の塊（ステップ）ごとに非表示にする。
* 画面下部に固定（またはコンテンツ下部）の「次へ進む」ボタンを配置する。
* ボタンを押すたびに、次のテキストや画像がフェードイン等のアニメーション付きで表示される。
* 章の最後に到達したら、ボタンの表示を「次の章へ」に切り替え、次章へ遷移させる。


* **インタラクティブなワーク:**
* Markdown内の「ワーク」「記述欄」に該当する箇所には、HTMLの `<textarea>` や `<input>` を配置する。
* ユーザーがブラウザ上で直接考えを書き込めるようにする（保存機能は不要）。


* **【追加要件】まとめスライド（PDF）表示機能:**
* **配置:** 各章の最後のステップが表示された後に、「まとめスライドを表示」ボタンを表示する。
* **挙動:** ボタンを押下すると、画面全体を覆うオーバーレイ（モーダル）が表示され、その中でPDFを表示する。
* **ファイル参照先:** `index.html` と同階層にある `slide` フォルダ内のPDFファイルを参照する（例: 1章なら `slide/chapter1.pdf`、2章なら `slide/chapter2.pdf` のように命名規則を想定したコードにする）。
* **閉じる機能:** オーバーレイ内に「閉じる」ボタンを配置し、押下すると元の講義画面に戻れるようにする。



## 4. コンテンツの反映

* 添付の `向上心.md` の内容を漏らさず実装すること。
* Markdownの階層構造（# 章, ## 節, ### 項）をHTMLのセクション構造に適切に変換すること。
* 数式が含まれる場合は、MathJax等を用いて綺麗にレンダリングされるようにすること。

## 出力のお願い

解説は最小限にし、すぐに動作確認ができる `index.html`, `style.css`, `script.js` の3つの完全なソースコードを提示してください。

# Docker習得カリキュラム（全8時間）

## 全体スケジュール概要

| セクション | 時間 | 内容 | ゴール |
| --- | --- | --- | --- |
| **1. 基礎** | 2時間 | Dockerの概念と基本操作 | コンテナを起動・操作できる |
| **2. 応用(前半)** | 3時間 | 仕組みの理解とイメージ作成 | Dockerfileで自作イメージを作る |
| **3. 応用(後半)** | 3時間 | 3層アプリ構築実践 | 複数コンテナを連携させ環境を作る |

---

## 1. 基礎編：Dockerに触れてみよう（2時間）

ここでは、座学は最小限にし、実際に手を動かして「コンテナが動く」体験を重視します。

### 1-1. Dockerの基礎概念（30分）

* **Dockerとは**
* コンテナ型仮想化の概要
* 仮想マシン（VM）とコンテナの違い


* **Dockerの目的**
* 「環境の差異」をなくす（Works on my machine問題の解決）
* IaC（Infrastructure as Code）の第一歩



### 1-2. 環境構築（45分）

* **Dockerのインストール（ハンズオン）**
* Docker Desktop (Windows/Mac) または Docker Engine (Linux) のインストール
* 動作確認：バージョン確認コマンドの実行



### 1-3. 基本コマンドとライフサイクル（45分）

* **Hello World（ハンズオン）**
* `docker run hello-world` の実行と出力結果の確認


* **Webサーバ起動（ハンズオン）**
* Nginxコンテナの起動：`docker run -d -p 8080:80 nginx`
* ブラウザで `localhost:8080` にアクセスし、Nginxの画面を確認する


* **コンテナ操作（ハンズオン）**
* コンテナの一覧表示 (`ps`)
* 停止 (`stop`)、開始 (`start`)
* 削除 (`rm`)
* イメージの確認と削除 (`images`, `rmi`)

## 2. 応用編：仕組みの理解と構築（6時間）

単にコマンドを打つだけでなく、裏側の仕組みを理解し、複数のコンテナを組み合わせてサービスを構築する力を養います。

### 2-1. Dockerの仕組みとDocker Hub（1時間30分）

* **Dockerの仕組みを詳しく知る**
* **Dockerコンポーネント**: Docker Daemon, Docker Client, Registryの関係性
* **イメージのレイヤ構造**: 差分管理の仕組み（なぜDockerは軽いのか）


* **Docker Hubの利用**
* **Docker Hubとは**: 公式レジストリの役割
* **Docker Hubでできること**: イメージの検索、Push/Pull
* **Docker Hubの利用（ハンズオン）**:
* Web上で自分の作りたい環境（例: Python, MySQL）の公式イメージを検索してみる
* タグ（バージョン）の読み方を理解する





### 2-2. Dockerfileによるイメージ作成（1時間30分）

3層アプリを作るには、既存のイメージを使うだけでなく、アプリケーションコードを含んだ「独自のイメージ」を作る必要があります。

* **Dockerfileの基本**
* `FROM`, `RUN`, `COPY`, `CMD` 命令の理解


* **ハンズオン：オリジナルWebサーバを作る**
* 簡単なHTMLファイルを作成
* Nginxをベースに、自作HTMLをコピーしたイメージをビルド（`docker build`）
* 自作イメージからコンテナを起動確認



### 2-3. 3層アプリケーションの環境構築（3時間）

ここが最終ゴールです。**Docker Compose**（複数コンテナの定義ツール）を導入し、Web/App/DBを連携させます。
※コマンド単体で3層連携は複雑すぎるため、実務で必須となるDocker Composeを使用します。

* **構成概要**
* **Web**: Nginx（リバースプロキシとして動作）
* **App**: Python (Flask) または Node.js（簡単なAPIサーバ）
* **DB**: MySQL または PostgreSQL


* **ステップ1：Docker Composeの導入**
* `docker-compose.yml` の書き方（version, services, networks, volumes）


* **ステップ2：データベース層の構築（DB）**
* MySQLコンテナの定義
* 環境変数でのパスワード設定
* **データの永続化（Volume）**の設定（コンテナを消してもデータが消えないようにする）


* **ステップ3：アプリケーション層の構築（App）**
* 簡単なプログラムの作成（DBへの接続確認用コード）
* Dockerfileの作成とComposeへの組み込み


* **ステップ4：Webサーバ層の構築（Web）**
* Nginxの設定ファイル（`default.conf`）作成
* Appコンテナへの転送設定（リバースプロキシ）


* **最終確認**
* `docker-compose up` で一括起動
* ブラウザからアクセスし、DBからデータが取得できるか確認
* `docker-compose down` で環境の片付け
